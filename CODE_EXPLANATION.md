# توضیح کامل کد — ESP8266 Dual LED Blink

این فایل تمام کدهای پروژه را خط به خط توضیح می‌دهد.

---

## فایل `platformio.ini`

این فایل تنظیمات محیط کامپایل و آپلود را مشخص می‌کند. PlatformIO قبل از هر چیز این فایل را می‌خواند.

```ini
[env:esp8266mod]
```

`[env:نام]` یک محیط (environment) تعریف می‌کند. می‌توانید چندین محیط برای بردهای مختلف داشته باشید. اینجا نام `esp8266mod` انتخاب شده — این نام دلبخواهی است و فقط برای شناسایی است.

---

```ini
platform = espressif8266
```

مشخص می‌کند از کدام toolchain استفاده شود. `espressif8266` مجموعه ابزارهای کامپایل مخصوص ESP8266 (کامپایلر xtensa-lx106-elf-gcc) را نصب و مدیریت می‌کند.

---

```ini
board = nodemcuv2
```

نوع دقیق برد سخت‌افزاری را مشخص می‌کند. `nodemcuv2` به PlatformIO می‌گوید:
- فرکانس CPU: 80 MHz
- حجم flash: 4MB
- نقشه پین‌ها: D0=GPIO16، D1=GPIO5، ...

> **نکته:** نام `esp8266mod` اشتباه است و کار نمی‌کند. نام صحیح `nodemcuv2` است.

---

```ini
framework = arduino
```

مشخص می‌کند از کدام فریم‌ورک استفاده شود. با `arduino`، توابع آشنا مثل `setup()`، `loop()`، `digitalWrite()`، و `delay()` در دسترس هستند.

---

```ini
upload_port = /dev/ttyACM0
monitor_port = /dev/ttyACM0
```

پورت سریالی که ESP8266 به آن وصل است. روی Raspberry Pi معمولاً `/dev/ttyACM0` است (بسته به تراشه USB-to-Serial ممکن است `/dev/ttyUSB0` باشد).

- `upload_port`: پورت برای آپلود firmware
- `monitor_port`: همان پورت برای خواندن خروجی Serial Monitor

---

```ini
monitor_speed = 115200
```

سرعت ارتباط سریال (baud rate). در صورت استفاده از `Serial.begin()` در کد، باید این مقدار یکسان باشد.

---

## فایل `src/main.cpp`

```cpp
#include <Arduino.h>
```

کتابخانه اصلی فریم‌ورک Arduino را include می‌کند. این کتابخانه توابعی مانند `pinMode`، `digitalWrite`، و `delay` را فراهم می‌کند. در PlatformIO این خط الزامی است (در Arduino IDE خودکار اضافه می‌شود).

---

```cpp
#define LED1 D0   // GPIO16
#define LED2 D1   // GPIO5
```

`#define` یک ثابت متنی تعریف می‌کند — نه متغیر. قبل از کامپایل، preprocessor هر جا `LED1` را ببیند با `D0` جایگزین می‌کند.

- `D0` روی برد NodeMCU برچسب‌گذاری شده و در واقع `GPIO16` است
- `D1` همان `GPIO5` است
- استفاده از `#define` به جای عدد مستقیم کد را خواناتر می‌کند و تغییر پین را آسان‌تر

---

```cpp
void setup() {
```

این تابع **یک بار** در ابتدای اجرا فراخوانی می‌شود. مناسب برای تنظیمات اولیه مثل پین‌ها، Serial، و متغیرها.

---

```cpp
  pinMode(LED1, OUTPUT);
  pinMode(LED2, OUTPUT);
```

جهت هر پین را مشخص می‌کند:

- `OUTPUT` — پین می‌تواند ولتاژ تولید کند (برای کنترل LED)
- `INPUT` — پین ولتاژ را می‌خواند (برای دکمه یا سنسور)

اگر `pinMode` را فراموش کنید، پین در حالت `INPUT` با مقاومت شناور (floating) می‌ماند و رفتار غیرقابل پیش‌بینی دارد.

---

```cpp
void loop() {
```

این تابع **به صورت بی‌نهایت** و پشت سر هم اجرا می‌شود — مثل یک حلقه `while(true)` که ESP8266 تا زمانی که برق دارد اجرا می‌کند.

---

```cpp
  digitalWrite(LED1, HIGH);  // LED1 روشن
  digitalWrite(LED2, LOW);   // LED2 خاموش
  delay(1000);               // یک ثانیه صبر
```

**`digitalWrite(pin, value)`** — ولتاژ پین را تنظیم می‌کند:
- `HIGH` = ۳.۳ ولت → اختلاف ولتاژ از پین به GND → جریان از LED عبور می‌کند → LED روشن
- `LOW` = ۰ ولت → اختلاف ولتاژی نیست → جریانی نیست → LED خاموش

**`delay(1000)`** — اجرا را ۱۰۰۰ میلی‌ثانیه (۱ ثانیه) متوقف می‌کند.

> **هشدار:** `delay()` CPU را کاملاً مسدود می‌کند و در این مدت هیچ کار دیگری انجام نمی‌دهد. در پروژه‌های پیچیده‌تر از `millis()` استفاده کنید تا CPU آزاد بماند.

---

```cpp
  digitalWrite(LED1, LOW);   // LED1 خاموش
  digitalWrite(LED2, HIGH);  // LED2 روشن
  delay(1000);               // یک ثانیه صبر
}
```

در نیمه دوم حلقه وضعیت LEDها برعکس می‌شود. این چرخه به صورت بی‌نهایت تکرار می‌شود.

## مفاهیم کلیدی برای یادگیری بیشتر

- **GPIO (General Purpose Input/Output):** پین‌های چندمنظوره که می‌توانند ورودی یا خروجی باشند
- **PWM (Pulse Width Modulation):** روشی برای شبیه‌سازی خروجی آنالوگ با پالس‌های دیجیتال — برای کنترل روشنایی LED
- **Serial Communication:** ارتباط متنی با ESP8266 از طریق پورت سریال برای debug
- **millis():** جایگزین `delay()` که CPU را آزاد می‌گذارد و چند کار را به صورت همزمان امکان‌پذیر می‌کند
